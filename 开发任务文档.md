# 棋牌计分小程序开发任务文档（Coding TODO）

## 0. 当前约束（已确认）
- 房间状态仅有：`open`、`closed`
- 房间允许任意时刻人员加入与退出（不受对局状态限制）
- 对局内只做手动计分，不实现积分规则引擎
- 不设置“回合”概念，按记分流水记账（转分/单人加分）
- 后端框架：Gin
- 数据库：MongoDB（本清单先按接口与结构推进，数据库实现可后置）

## 1. 后端任务拆分（按优先级）

### P4 - Room 模块（核心）
- [ ] `POST /api/rooms` 创建房间（返回 roomCode）
- [ ] `POST /api/rooms/join` 加入房间（房间为 `open` 才可加入）
- [ ] `POST /api/rooms/leave` 主动离开房间
- [ ] `GET /api/rooms/{roomCode}` 获取房间详情
- [ ] `POST /api/rooms/{roomCode}/close` 关闭房间
- [ ] 处理并发加入同一房间的幂等与去重

### P5 - Match 模块（计分 MVP）
- [ ] `POST /api/matches` 基于 roomCode 创建对局

- [ ] `POST /api/matches/{matchId}/grant` 记录单用户加分
- [ ] `PATCH /api/matches/{matchId}/entries/{entryNo}` 修改记分流水
- [ ] `POST /api/matches/{matchId}/finish` 结束对局
- [ ] `GET /api/matches/history` 查询历史对局（分页）

## 2. 建议编码顺序（专注 MVP）
1. P0 + P2（先把服务跑起来并锁定接口契约）
2. P4（先打通房间创建/加入/退出/查询）
3. P5（再实现对局计分主流程）
4. P3（补齐微信登录鉴权）
5. P6 + P7（完善存储与测试）
